# Thread Journey — Artifact Compass Deep Salvage Guide

This document preserves the working journey from the Twis Holo Workshop / Road-Signal / Artifact Compass thread as source material for future Workshop design.

It is not runtime code.
It does not change app behavior.
It is a salvage guide: what happened, what matters, what should be preserved, what should become searchable, and what should wait.

## One-line thesis

Twis Holo Workshop is becoming a human-controlled creative workshop: local-first, artifact-first, source-aware, proof-disciplined, music-capable, and guarded against shiny uncontrolled tool drift.

## Pass 1 — Debugged story of the journey

The user entered the thread carrying a scattered but coherent system: a private local Workshop, a phone/cloud field kit, a GitHub deploy shelf, and a need to keep creative work from scattering across tools, prompts, ZIPs, images, music, and half-finished ideas.

The first meaningful act was boundary-setting.

The Workshop was explicitly protected from becoming:

- a full DAW;
- a cloud-private storage trap;
- a paid-API dependency stack;
- an MCP nervous system;
- a browser-agent access mess;
- an untested agentic dashboard;
- a shiny UI that outruns working rooms.

That boundary mattered because the project already had strong gravitational pull toward bloat. Every new tool, model, UI pattern, and creative feature could have become another half-built system. The user kept forcing the work back to one question:

> What is useful, what is proven, and what belongs later?

The repo pass then confirmed existing state instead of inventing it:

- Road-Signal Machine v3 is active in the Music room.
- Road-Signal can build music prompt artifacts.
- The artifact-save bridge exists.
- The bridge is wired into `app/index.html`.
- Browser mode saves to localStorage.
- Local companion sync is attempted but not proven until tested locally.
- Cloudflare AI already uses `@cf/openai/gpt-oss-20b` and should not be patched again.

The first code-facing result was intentionally boring and correct: documentation-only guardrail docs.

- `WORKSHOP_ARCHITECTURE.md`
- `SECURITY_GATES.md`
- `CLOUDFLARE_COST_GUARDRAILS.md`
- README links

No runtime behavior changed.
No dependencies were added.
No hosted agents were added.
No new feature was started.

That was the Workshop learning not to lie.

## Pass 3 — Proof / trust salvage

The user then added the FlashRiver Phase 12 harness extraction. That material gives the Workshop its proof discipline.

Salvaged proof/trust patterns:

- prove actual file access first;
- keep claims narrow;
- certify claims, not tools;
- never broaden a certificate beyond evidence;
- classify work as plan, audit, proof, repair, gate, or release;
- treat metadata/UI/cards as non-executable unless separately implemented and proven;
- quarantine failure honestly;
- repair the smallest proven defect;
- preserve working-memory files as continuation truth.

The principle:

> What exact claim are we making, what exact evidence supports it, and what are we forbidden to assume?

This applies to:

- repo changes;
- local file access;
- imported artifacts;
- MCP candidates;
- Cloudflare endpoints;
- AI model claims;
- Artifact Compass results;
- Windows setup claims;
- SQLite/disk sync claims;
- future agentic UI claims.

## Pass 4 — Music salvage

The Music room became clearer in this thread.

It is not a full DAW.
It is a Road-Signal song command bench.

Correct shape:

```text
capture idea
-> choose sound identity
-> choose gear stack
-> write lyric block
-> build Producer AI prompt
-> build fused-line prompt
-> build code-shaped packet
-> save artifact
-> find it later in My Work / Artifact Compass
```

Road-Signal identity:

- older deep gravel male voice;
- rapid-fire spoken-word bursts;
- female AI / echo countervoice;
- deep house / tribal pulse;
- tom-heavy percussion;
- dirty guitar;
- ghost cello and violin;
- banjo wire ticks;
- static, scratch, flanger, phaser, tape delay;
- rain, road hum, dog-door texture, old truck rhythm, workshop-machine grit;
- no coffee imagery;
- beer / road / wire / dogs / trucks / busted machines are allowed.

Main style families salvaged:

- Road-Signal Deep House;
- Tom-Engine Tribal;
- Industrial Blues / Machine Folk;
- Dark Cinematic Electronic;
- Hard Rock Hybrid;
- Orchestral Road Signal;
- Glitch / Static / Broken Radio;
- Desert / Backroad / Americana Electronic.

Key mechanism:

> Same Road-Signal soul, different swappable rhythm and instrument gears.

This prevents same-song disease.

## Pass 5 — UI / cockpit salvage

The user noticed the UI felt too plain.

The important salvage was not immediate UI implementation. It was future direction:

- richer typography;
- stronger visual hierarchy;
- glass/depth treatment;
- asymmetric card layout;
- status-aware cards;
- embedded prompt/action zones;
- agentic-feeling interface without autonomous agent behavior.

Core principle:

> Make it feel agentic before making it truly agentic.

Future UI should feel like:

- Road-Signal Cockpit;
- Signal Bench;
- Creative command system;
- future-retro dark glass workspace;
- local-first human-controlled dashboard.

It must not become:

- an autonomous agent panel;
- MCP control center;
- cloud sync trap;
- browser agent with localhost access;
- flashy dashboard before room workflows work.

## Keep / Adapt / Park / Reject

### Keep

- Local/cloud/GitHub authority boundary.
- Road-Signal music identity.
- Artifact-save workflow as current Music room priority.
- Creative-control source packets.
- FlashRiver proof/trust source packet.
- Windows setup instructions.
- The journey narrative as user-facing Workshop identity material.

### Adapt later

- Direction Cards into first-class Workshop artifacts.
- Context Packs into reusable pre-generation cards.
- Style Matrices into image/video/music/UI design controls.
- Scorecards into review workflows.
- Proof/Trust cards into file/tool/cloud validation workflows.
- Road-Signal palette into Music room preset selectors.
- Future-retro cockpit UI into a bounded CSS/UI pass.

### Park

- Runtime enforcement of proof/trust rules.
- Full agentic UI.
- MCP integrations.
- Browser agents.
- Cloud private sync.
- ComfyUI/custom-node hosting.
- Local voice model integration.
- Full DAW behavior.

### Reject

- Claiming SQLite/disk sync is proven before local test.
- Treating cloud as private authority.
- Treating GitHub as private data storage.
- Treating free/open-source tools as trusted by default.
- Adding paid APIs as core dependencies.
- Letting UI imply enforcement/certification it does not perform.
- Generic AI music/image language that erases the user's actual imagery.
